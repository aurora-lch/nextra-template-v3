# helm 管理

Helm 是一个 Kubernetes 的包管理器，用于查找、共享和使用在 Kubernetes 上运行的软件包。

![Mr04b1N0uoMN7rxVc4Vcl7jJn6d](/static/Mr04b1N0uoMN7rxVc4Vcl7jJn6d.png)

## **使用 Helm 管理页面的方法**

- **选择集群和命名空间**：在页面顶部选择适当的 Kubernetes 集群和命名空间。
- **查看 Helm 列表**：浏览当前集群和命名空间下的 Helm 应用列表。
- **执行操作**：选择特定 Helm 应用并执行需要的操作。

| **操作** | **描述**                          |
| -------- | --------------------------------- |
| **详情** | 查看 Helm 应用的详细信息。        |
| **升级** | 更新 Helm 应用到 Chart 的新版本。 |
| **回滚** | 将 Helm 应用回滚到之前的版本。    |
| **删除** | 完全移除 Helm 应用。              |

- **创建新的 helm 应用**：点击【创建】按钮，按照向导部署新的 Helm 应用。

## **创建 helm 的步骤**

1. **集群分区：**在页面上选择应用将要部署的 Kubernetes 集群和将要存放的命名空间。
2. **选择 Chart：**

| **配置项**              | **描述**                                                     |
| ----------------------- | ------------------------------------------------------------ |
| **从仓库中选择 Chart**  | Helm 仓库是一个存储 Chart 的集中位置，可以是远程服务器或本地存储。 |
| **从本地上传 Chart 包** | 直接将 Chart 包文件上传到 Helm 客户端，而不是从远程仓库下载。 |

3. **部署服务**：输入服务名称后点击【部署】完成 helm 的创建。
