# Operator 管理

在 Kubernetes 生态系统中，Operator 是一种用于自动化部署、管理和升级复杂应用程序的机制。

## Operator 产品列表页面

![ZXn2bU1gNoqVsAxARalcUwB3nBf](/static/ZXn2bU1gNoqVsAxARalcUwB3nBf.png)

## **使用方法**

- **浏览 Operator 数据**：在【Operator 产品列表】页面，查看所有的 Operator 列表。
- **筛选 Operator：**可根据列表上方搜索栏的【选择集群】，点击【查询】，查看特定 Kubernetes 集群中的 Operator 列表。
- **管理 Operator**：点击 Operator 的【详情】按钮可查看当前 Operator 的基本信息和 Crd，或点击【删除】按钮来删除 Operator。
- **部署 Operator：**在【Operator 产品详情】页面，点击【部署 Operator】按钮，按照向导来部署 Operator。

## **创建关联**

在【Operator 产品列表】页面，点击【创建关联】按钮，按照向导为 Operator 创建关联。

### 1. 选择 Chart

| **配置项**              | **描述**                                                     |
| ----------------------- | ------------------------------------------------------------ |
| **从仓库中选择 Chart**  | Operator 仓库是一个存储 Chart 的集中位置，可以是远程服务器或本地存储。 |
| **从本地上传 Chart 包** | 直接将 Chart 包文件上传到 Operator 客户端，而不是从远程仓库下载。 |

### 2. 集群分区

| **配置项**   | **描述**                         |
| ------------ | -------------------------------- |
| **租户**     | 从下拉列表中选择一个特定的租户。 |
| **订单**     | 选择与租户相关的订单。           |
| **集群**     | 选择租户使用的 Kubernetes 集群。 |
| **命名空间** | 在集群内选择特定的命名空间。     |
