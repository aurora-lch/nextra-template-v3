# 命名空间管理

命名空间是 Kubernetes 集群中的一个逻辑分区，用于隔离资源，提供资源访问的多租户环境。用于将集群资源分配给不同的用户、团队或项目，以实现资源的逻辑隔离和管理。

## **使用步骤**

- **浏览命名空间**：在页面上查看列出的所有命名空间。如有多页，点击右下方翻页按钮 \<、> 或具体页数，可查看对应页面信息。
- **筛选命名空间：**可通过在列表上方搜索栏输入分区名称，或在下拉列表中选择集群，点击“搜索”，查看特定的命名空间。
- **删除命名空间：**通过【操作】点击【删除】按钮来删除对应的命名空间。
- **管理命名空间**：从集群列表中选择对应的命名空间进行管理，通过【操作】点击【配置】按钮可对相应的命名空间进行配置。

1. 修改命名空间的资源配额管理（ResourceQuota）

| **资源类型**                   | **描述**                                                     |
| ------------------------------ | ------------------------------------------------------------ |
| **CPU 初始值**                 | resources resquest cpu，所有非终止状态的 Pod，初始需求的 CPU 数量。若未设置则默认使用 CPU 上限值。 |
| **内存初始值**                 | resources resquest memory，所有非终止状态的 Pod，初始需求的内存数量。若未设置则默认使用内存上限值。 |
| **CPU 上限**                   | resources limits cpu，非终止状态的 Pod，其 CPU 限额总量不能超过该值。（注：如果配置了初值，那么上限值不能低于初值） |
| **内存上限**                   | resources limits memory，非终止状态的 Pod，其内存限额总量不能超过该值。（注：如果配置了初值，那么上限值不能低于初值） |
| **Pod 数量**                   | 在该命名空间中允许存在的非终止状态的 Pod 总数上限。          |
| **service 数量**               | 在该命名空间中允许存在的 Service 总数上限。                  |
| **RC 数量**                    | 在该命名空间中允许存在的 ReplicationController 总数上限。    |
| **资源限制数量**               | 在该命名空间中允许存在的 ResourceQuota 总数上限。            |
| **secret 数量**                | 在该命名空间中允许存在的 Secret 总数上限。                   |
| **persistentvolumeclaim 数量** | 在该命名空间中允许存在的 PVC 的总数上限。                    |

2. 修改容器资源限制默认值管理（LimitRange）

| **资源类型**  | **描述**                                          |
| ------------- | ------------------------------------------------- |
| CPU 上限      | 非终止状态的 container，其 CPU 限额不能超过该值。 |
| 内存上限      | 非终止状态的 container，其内存限额不能超过该值。  |
| CPU 初值      | 非终止状态的 container，初始需求的 CPU 数量。     |
| 内存初值      | 非终止状态的 container，初始需求的内存数量。      |
| 默认 CPU 上限 | 用户未设定应用容器的 CPU 上限时，则使用该值。     |
| 默认内存上限  | 用户未设定应用容器的内存上限时，则使用该值。      |
| 默认 CPU 初值 | 用户未设定应用容器的 CPU 初值时，则使用该值。     |
| 默认内存初值  | 用户未设定应用容器的内存初值时，则使用该值。      |
