# 集群管理

支持对多个 Kubernetes 集群的统一管理，便于在不同的网络域和数据中心之间部署和管理应用。支持集群状态监控、节点数据、namespace 信息的管理。

![Mv37bNUXCo6s1Sx4bnqc0BUZnXg](/static/Mv37bNUXCo6s1Sx4bnqc0BUZnXg.png)

## **使用方法**

- **浏览集群**：在【集群管理】页面，查看所有的集群信息及健康状态。
- **筛选集群：**可通过在列表上方搜索栏输入集群名称，点击【搜索】，查找特定的集群。
- **管理集群**：从集群列表中选择对应的集群进行管理，通过【操作】点击【详情】按钮可查看对应集群的

集群概览、主机列表、Namespace 和安全配置，或点击【移除】按钮来移除集群。

## **纳管集群**

在【集群管理】页面，点击【纳管集群】进入纳管集群向导页面。

| **配置项**                                 | **描述**                                                     |
| ------------------------------------------ | ------------------------------------------------------------ |
| **基本信息**                               | **集群名称**：必填。输入集群的名称。**IPv4/IPv6**：当业务应用涉及到以下场景时，建议启用双栈：- 应用需要为使用 IPv6 终端的用户提供访问服务。- 需要对使用 IPv6 终端访问应用提供的服务的访问来源进行数据分析处理。- 应用系统与其他系统、应用系统之间需要使用 IPv6 进行内网访问。**GPU 集群**：默认关闭。如需要请开启。 |
| **容器网络**                               | **Cluster CIDR**：供集群创建时启动的容器组使用的 IP 地址段,需填写集群上配置的 Cluster CIDR。**Service CIDR**：供类型为 ClusterIP 的 Kubernetes Service 使用的 IP 地址段，需填写集群上配置的 cluster CIDR，不可与 Cluster CIDR 的网段重叠。**helm apiserver 地址**：输入 helm apiserver 的地址。 |
| **K8s Master（默认使用 Calico 网络策略）** | **Master 地址**：输入 Kubernetes Master 节点的地址。**Ca 私钥**：需要输入 CA（证书颁发机构）的私钥，私钥应以"-----BEGIN RSA PRIVATE KEY-----"开头，并以"-----END RSA PRIVATE KEY-----"结束。**Ca 证书**：需要输入 CA 证书，证书应以"-----BEGIN CERTIFICATE-----"开头，并以"-----END CERTIFICATE-----"结束。 |
| **私库**                                   | **私库地址**：从下拉列表中选择私有镜像库的地址。             |
| **etcd**                                   | **etcd 地址**：输入 etcd 节点的地址，如果有多个 etcd 地址，使用“新增 Etcd 地址”按钮添加。 |
| **etcd 私钥与证书**                        | **etcd 私钥(ca.key)**：用于 etcd 节点间通信的非对称加密密钥。**etcd 证书(ca.crt)**：用于 etcd 节点身份验证的数字证书，通常与私钥配对使用。 |
